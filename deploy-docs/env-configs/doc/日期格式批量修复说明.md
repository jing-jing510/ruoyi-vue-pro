# Coal模块日期格式批量修复说明

## 修改日期
2025-12-25

## 问题描述
Coal模块中的多个表单组件使用了错误的日期格式 `value-format="x"`（毫秒时间戳），导致提交时出现日期解析错误：
```
Invalid value for Year (valid values -999999999 - 999999999): 4835110699
```

## 问题原因
- 前端使用 `value-format="x"` 发送毫秒时间戳
- 后端 `LocalDate` 类型无法解析毫秒时间戳（只能解析日期字符串）
- 后端 `LocalDateTime` 类型虽然可以接收时间戳，但使用字符串格式更规范

## 修复规则

### 日期格式对照表
| 后端类型 | 前端type | 前端value-format | 说明 |
|---------|---------|-----------------|------|
| LocalDate | date | YYYY-MM-DD | 只有日期 |
| LocalDateTime | datetime | YYYY-MM-DD HH:mm:ss | 日期+时间 |

### 修复步骤
1. 查看后端DO对象中的字段类型
2. 根据类型修改前端日期选择器的配置：
   - `LocalDate` → `type="date"` + `value-format="YYYY-MM-DD"`
   - `LocalDateTime` → `type="datetime"` + `value-format="YYYY-MM-DD HH:mm:ss"`

## 常见错误

### 错误1：type和value-format不匹配
```vue
<!-- 错误：type是date但format包含时间 -->
<el-date-picker type="date" value-format="YYYY-MM-DD HH:mm:ss" />

<!-- 正确：type和format匹配 -->
<el-date-picker type="date" value-format="YYYY-MM-DD" />
<el-date-picker type="datetime" value-format="YYYY-MM-DD HH:mm:ss" />
```

### 错误2：后端类型和前端格式不匹配
```vue
<!-- 错误：后端是LocalDate但发送时间戳 -->
<el-date-picker type="date" value-format="x" />  <!-- 后端: LocalDate -->

<!-- 正确：后端是LocalDate发送日期字符串 -->
<el-date-picker type="date" value-format="YYYY-MM-DD" />  <!-- 后端: LocalDate -->
```
